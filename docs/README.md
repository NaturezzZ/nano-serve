# Documentation Index

Start here when planning or implementing a feature.

- [Architecture](architecture.md)
- [Benchmarking](benchmarking.md)
- [Roadmap](roadmap.md)
- [References](references.md)
- [Feature design docs](features/README.md)

Documentation rules:

- Every roadmap feature must have a document in `docs/features/`.
- A feature document must be updated in the same change that modifies the
  feature behavior.
- Design docs should describe tests and benchmarks before implementation
  details.
- If the implementation disproves the design, update the document instead of
  leaving stale plans behind.
